# fitlog — design system

> Provisional v0. Final shape decisions per-screen go through `/impeccable shape` before code. This file documents the system tokens once a screen is built; update as the system stabilizes.

## Scene

A self-coached lifter glances at their phone between sets in a noisy commercial gym at 7pm. Bright overhead lights, sweat on the screen, one-handed in the right thumb, sometimes the phone is propped between dumbbells. They have ~15 seconds to read previous-session weights, log a set, and start a rest timer before their next working set.

The UI must be readable at arm's length, glanceable while breathing hard, and not pull attention away from the lift.

## Color

**Strategy:** Restrained. Tinted dark neutrals carry the surface, a single high-chroma lime carries focus and progress. Lime is ≤10% of any screen — it lights up the **next thing the user is about to interact with** (the active set, the rest-timer, a PR badge), nothing else.

All colors in OKLCH. Reduce chroma at the lightness extremes.

### Tokens (dark, default)

| Role | OKLCH | Notes |
|---|---|---|
| `--bg` | `oklch(0.16 0.005 150)` | near-black, hint of green |
| `--bg-elev-1` | `oklch(0.20 0.006 150)` | card surface |
| `--bg-elev-2` | `oklch(0.24 0.007 150)` | sheet / popover |
| `--text` | `oklch(0.94 0.005 150)` | primary text |
| `--text-mute` | `oklch(0.68 0.008 150)` | secondary text |
| `--text-dim` | `oklch(0.52 0.008 150)` | tertiary, hints |
| `--border` | `oklch(0.30 0.008 150)` | hairline 1px |
| `--accent` | `oklch(0.86 0.20 130)` | lime — interactive focus, in-progress, PR |
| `--accent-mute` | `oklch(0.86 0.20 130 / 0.12)` | lime tinted bg for active state |
| `--ok` | `oklch(0.78 0.16 145)` | set-complete, slightly desaturated lime variant |
| `--warn` | `oklch(0.80 0.15 70)` | amber, used sparingly (rest overrun) |
| `--danger` | `oklch(0.65 0.20 25)` | only for delete-account confirms |

### Tokens (light)

Generated by inverting the lightness scale; lime drops chroma to `0.15` so it doesn't burn on a white surface. Light mode is opt-in (settings), not the default.

### Bans

- No `#000` or `#fff`. Every neutral tints toward green at chroma ~0.005–0.008.
- No gradient text. No glassmorphism. No side-stripe-borders. No big-number-with-gradient cliché.

## Typography

**Stack:** system fonts only — no Google Fonts, no FOIT/FOUT. Speed is a feature.

```css
--sans: -apple-system, BlinkMacSystemFont, 'Inter', 'SF Pro Text', 'Segoe UI',
        Roboto, 'Helvetica Neue', sans-serif;
--mono: ui-monospace, 'SF Mono', 'Cascadia Mono', 'JetBrains Mono', monospace;
```

**Numerics use `font-variant-numeric: tabular-nums`** so columns of weights and reps don't dance during edits — non-negotiable in the logger.

**Scale (rem on 1rem = 16px base):**

| Token | rem | use |
|---|---|---|
| `--t-xs` | 0.75 | meta, timestamps |
| `--t-sm` | 0.875 | secondary labels |
| `--t-md` | 1.0 | body |
| `--t-lg` | 1.125 | section headings |
| `--t-xl` | 1.5 | screen titles |
| `--t-2xl` | 2.25 | metric values (e.g., "85 kg") |
| `--t-3xl` | 3.5 | hero metrics (rest-timer, streak count) |

Line height: 1.4 for body, 1.0 for metrics.

**Weight contrast** ≥1.5: body 400, headings 600, metric values 700. Never below 400.

Body line length capped at 65ch — mostly affects `Settings` + onboarding; logger has no body copy.

## Spacing

4px grid. Tokens `--s-1` (4px) through `--s-12` (48px). Vary spacing for rhythm — section gaps are 32px, set-row gap is 8px. Never every-component-same-padding monotony.

## Shape

| Token | px | use |
|---|---|---|
| `--r-sm` | 6 | inputs, chips |
| `--r-md` | 12 | small cards |
| `--r-lg` | 20 | primary cards (workout, routine) |
| `--r-xl` | 28 | bottom sheets |
| `--r-pill` | 999 | streak ring, primary CTA |

**Card rule:** flat surface, no nested cards. Elevation comes from `--bg-elev-1` background lift on a flat hairline border, **not from drop shadows**. One soft shadow only on the bottom-nav (to feel sticky over scrolled content).

## Motion

- Easing: ease-out-quart `cubic-bezier(0.165, 0.84, 0.44, 1)` for transitions.
- Spring on set-complete: `framer-motion` `{ type: 'spring', stiffness: 380, damping: 28 }`.
- Page transitions: 180ms fade + 8px y-offset. Never animate `width`/`height`/`top`/`left`.
- Honor `useReducedMotion()` — drop all enter animations to instant for users who set it.
- Rest timer: numeric tick is a tabular re-render, not a CSS transition. Adding a transition makes it feel laggy.

## Components — key conventions

- **SetRow.** Inline numeric steppers for weight + reps. ✓ button on the right, full-row tap target. Active row gets `--accent-mute` background so the lifter's eye snaps to it at arm's length.
- **NumberStepper.** ± buttons sized 44×44 minimum, value in `tabular-nums`, long-press repeats.
- **RestTimer.** 3xl tabular numeric, full-width, colored `--accent` while running, drops to `--warn` if overrun by >30 sec. Big "+30s" / "skip" pill buttons under it.
- **Bottom nav.** 4 tabs, 56px height, icons + lowercase labels. Sticks to bottom safe-area inset. Sole shadow on the surface.
- **Cards.** No drop shadow. Hairline border + elevated background. Border radius `--r-lg`.

## Empty states

Never a generic mascot. Every empty state is **a single sentence + the one thing they should do next**. Example: "no workouts yet. start one →"

## Reflex check

First-order trap is **fitness tracker → dark + neon lime**. We've taken that on purpose because:
- the scene sentence forces dark (gym overhead lighting + screen reflection),
- lime ≤10% in a Restrained palette is not the saturated "neon dashboard" reflex — it's an accent doing the job of an accent,
- the absence of feed/profile/community gravity removes the second-order trap (no card-of-other-people, no avatar grid, no leaderboard rows).

If the system starts converging on stat-card-grids or hero-metric templates, treat it as a regression — those are the SaaS clichés the impeccable laws ban.
